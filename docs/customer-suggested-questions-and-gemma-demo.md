# Senticore 고객별 예상질문 10개 및 Gemma 시연 로그

## 목적

시연에서 상담원이 고객을 선택했을 때 AI스크립트 영역에 표시할 예상질문 후보와,
로컬 Gemma가 Senticore 상담 케이스에 대해 실제로 어떤 수준의 답변을 생성하는지 보여주기 위한 기록이다.

## Gemma 시연 프롬프트

모델: `gemma4:e4b`

호출 방식:

```bash
curl -s http://127.0.0.1:11434/api/generate \
  -d '{"model":"gemma4:e4b","stream":false,"think":false,"format":"json","options":{"temperature":0.1,"num_predict":800},"prompt":"..."}'
```

프롬프트 본문:

```text
Senticore 채권관리 상담 보조 AI로 답하라.
반드시 JSON만 출력하라.
모든 점수는 0부터 100 사이 정수다.
nextActions는 문자열 배열 3개다.
draft는 고객에게 직접 보낼 수 있는 한국어 3문장이다.
내부 점수, 전략명, 위험 라벨은 draft에 쓰지 않는다.
확정 승인, 감면 확약, 법적조치 단정은 금지한다.

출력 JSON 키:
intent, sentimentScore, cooperation, stability, riskCue, draft, notice, nextActions

고객:
박재원 / 차량담보대출 / D+124 / 회수등급 D / 건전성 고정 / 전화 거부 / 문자 선호 / 민원 위험.

문의:
법적조치가 정확히 언제 들어가나요? 전화는 받기 어렵고 문자로만 알려주세요.
지금 당장 낼 돈은 없습니다.
```

## Gemma 실제 응답

```json
{
  "intent": "상환 일정 문의 및 채무 조정 요청",
  "sentimentScore": 20,
  "cooperation": 30,
  "stability": 10,
  "riskCue": "법적조치 문의, 전화 거부, 상환 어려움 호소",
  "draft": "고객님, 현재 상황에 대해 어려움을 느끼시는 점 충분히 이해합니다. 법적 조치 시점과 관련해서는 구체적인 상담이 필요하오니, 문자 메시지로 편하신 시간을 알려주시겠어요? 저희가 가능한 최선의 상환 방안을 함께 모색해 보겠습니다.",
  "notice": "법적 조치 시점은 개별 채무 상황 및 내부 심사 기준에 따라 달라지므로, 정확한 안내는 상담을 통해 확인이 필요합니다. 문자 메시지 상담을 선호하시므로, 상담 가능 시간을 지정해 주시면 담당자가 연락드리겠습니다.",
  "nextActions": [
    "문자 상담 시간 재조율 요청",
    "채무 조정 가능성 안내",
    "상환 계획 재수립 제안"
  ]
}
```

## 판단

Gemma는 핵심 의도와 위험 신호를 잘 잡았다.

- 법적조치 문의를 주요 의도로 분류했다.
- 전화 거부와 상환 어려움을 위험 신호로 잡았다.
- 점수를 낮게 산정해 고위험/비협조 구간으로 판단했다.
- 고객에게 내부 점수나 위험 라벨을 직접 노출하지 않았다.

다만 운영 품질 기준에서는 보정이 필요하다.

- "편하신 시간을 알려주세요"는 전화 거부 고객에게 다소 어긋날 수 있다.
- "가능한 최선의 상환 방안"은 승인 가능성처럼 들릴 수 있어 더 보수적으로 다듬는 편이 낫다.
- Senticore 기준 답변은 "고지 이력 확인", "문자 채널 반영", "법적절차 단정 금지"를 더 명확히 둔다.

시연용 메시지:

> 로컬 Gemma도 고객 문의의 의도, 감성 위험, 다음 조치 후보를 잡을 수 있다. 다만 금융 상담 문구는 그대로 자동 발송하지 않고, Senticore의 준법 가드레일과 템플릿 보정으로 최종 상담 초안을 만든다.
