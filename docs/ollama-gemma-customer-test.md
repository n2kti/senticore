# Senticore Ollama Gemma 고객별 테스트

## 목적

로컬 Ollama의 `gemma4:e4b`만 사용했을 때 Senticore 상담 케이스에 대해 어느 정도 답변을 생성하는지 확인한 기록이다.
시연에서는 "로컬 모델도 상담 초안 생성은 가능하지만, 금융 상담 문구는 Senticore 보정 레이어를 거쳐야 한다"는 근거로 사용한다.

## 테스트 조건

- 모델: `gemma4:e4b`
- 호출: `http://127.0.0.1:11434/api/generate`
- 옵션: `stream=false`, `think=false`, `format=json`, `temperature=0.1`, `num_predict=800`
- 케이스: 고객 5명, 고객별 질문 2개씩 총 10건
- 출력 요구: `intent`, `sentimentScore`, `cooperation`, `stability`, `riskCue`, `draft`, `notice`, `nextActions`

## 전체 판단

10건 모두 JSON 응답 자체는 돌아왔다.
응답 시간은 약 13-31초였고, 평균은 약 19.5초 수준이다.

다만 운영 품질 기준으로는 보정이 필요하다.

- `nextActions`가 배열이 아니라 문자열로 나온 케이스가 3건 있었다.
- `nextActions`가 3개가 아니라 2개만 나온 케이스가 3건 있었다.
- 핵심 의도와 선호 채널은 대부분 잘 잡았다.
- 일부 문구는 "긍정적으로 검토"처럼 승인 가능성을 암시할 수 있어 준법 보정이 필요하다.

결론:

> Ollama Gemma는 로컬 시연용 상담 초안 생성 모델로는 사용 가능하다. 다만 라이브 주력 모델로 쓰려면 JSON 스키마 보정, nextActions 기본값 보충, 준법 문구 필터가 반드시 필요하다.

## 시연 추천 케이스

### 1. 오민석 - 새출발기금 접수 후 추심 중단 문의

추천 이유:

- 민감한 채무조정 이슈를 단정하지 않고 "서류/규정 확인"으로 처리했다.
- 이메일 선호 고객에게 맞는 응대 방향을 만들 수 있다.
- 로컬 모델이 규정성 질문도 초안 수준에서는 다룰 수 있음을 보여주기 좋다.

### 2. 최미래 - 정상납부 확인

추천 이유:

- 우호적 고객에게 과하게 딱딱하지 않은 답변이 나온다.
- 감성 점수와 협조 점수가 높게 나와 정상 고객 케이스 대비용으로 좋다.
- 위험 고객만 처리하는 서비스가 아니라 관계 유지 상담에도 쓸 수 있음을 보여준다.

### 3. 박재원 - 법적조치 시점 및 문자 안내 요청

추천 이유:

- 장기 연체, 전화 거부, 법적절차 불안이라는 고위험 신호를 잡았다.
- 법적조치 일정을 단정하지 않는 방향으로 답변했다.
- "이 답변을 Senticore 준법 템플릿으로 한번 더 보정한다"는 설명을 붙이기 좋다.

## 시연에서 피할 케이스

- `nextActions`가 문자열로 나온 케이스는 라이브 출력에서 형식 오류처럼 보일 수 있다.
- "긍정적으로 검토" 같은 표현이 들어간 케이스는 금융 상담에서 승인 기대를 줄 수 있어 그대로 보여주지 않는 편이 낫다.
- 응답이 30초 이상 걸린 케이스는 현장 네트워크나 PC 상태에 따라 지연이 더 커질 수 있다.

## 운영 보정안

로컬 Gemma를 붙일 경우 최소한 아래 보정이 필요하다.

1. JSON 파싱 실패 시 기본 응대 템플릿으로 대체
2. `nextActions`가 문자열이면 배열로 변환
3. `nextActions`가 3개 미만이면 고객 상태 기반 기본 액션으로 채움
4. `draft`에서 승인, 감면, 법적조치 확정처럼 들리는 표현 제거
5. 응답 시간이 20초를 넘으면 화면에 로컬 모델 처리 중 상태 표시
