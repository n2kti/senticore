# SentiCore AI 상담보조 IA 정의서 v2.0
> 작성일: 2026-05-18 | 기준 화면: React/Vite 데모 대시보드 | 목적: 시연/기획/후속 구현 기준 문서

---

## 1. 서비스 목적

SentiCore는 저축은행 채권관리 상담원이 고객과 통화하거나 문자/DM/이메일로 응대할 때, 고객의 연체 상태·계좌 특성·접촉 이력·감성 반응을 한 화면에서 보고 바로 상담 전략과 AI 스크립트를 만들 수 있게 하는 상담 보조 시스템이다.

AI는 최종 판단자가 아니다. AI는 상담원이 빠르게 판단할 수 있도록 고객 상태 요약, 행동 분석, 감성 분석, 응대 전략, 스크립트 초안을 제공한다. 법적 조치, 감면, 분할납부 승인, 채무조정 가능 여부는 반드시 담당자 확인 후 안내한다.

---

## 2. 현재 제품 방향

### 2.1 핵심 화면

현재 화면은 3열 고정 대시보드 구조다.

| 영역 | 역할 |
|---|---|
| 좌측 | `AI 상담보조 / SentiCore Assist`, 고객 접촉 현황, 상담 히스토리 |
| 중앙 | 고객 전환, 계좌 상태 요약, 타임 라인, 행동 분석, 감성 분석, 상담 전략 |
| 우측 | `AI스크립트`, 예상 질문, 직접 입력, LLM 호출 결과 |

### 2.2 주요 설계 원칙

- 전화 중 한눈에 들어오는 정보 밀도 유지
- 긴 설명보다 상태/판단/액션을 짧게 표시
- 상세보기는 보조 정보로 분리
- 고객별 카드 높이는 가능하면 고정
- 고객 전환 시 화면 위치와 레이아웃이 흔들리지 않아야 함
- 실제 LLM 호출 중에는 가짜 응답처럼 보이지 않도록 로딩 상태를 명확히 표시
- 예상 질문은 시연 안정성을 위해 고객별 preset으로 제공
- 직접 입력은 실제 LLM을 호출해 응답 품질을 확인

## 3. 화면 IA

### 3.1 좌측 패널: AI 상담보조

상단 타이틀:

```text
AI 상담보조  SentiCore Assist
```

주요 구성:

| 블록 | 내용 |
|---|---|
| 접촉 현황 | 최근 접촉 총 건수, 전화/문자/DM/이메일/방문별 접촉 수 |
| 채널 상태 | 전화/문자/DM/이메일/방문 가능 여부를 체크형 UI로 표시 |
| 히스토리 | 최근 상담, 문자, DM, 이메일, 방문 이력 |
| 스크롤 | 좌측 목록은 스크롤 가능해야 하며, 사용자에게 목록성 영역임이 보여야 함 |

히스토리에는 단순 상담 이력뿐 아니라 방문 이력도 포함한다. 전화 금지/문자 가능/DM 가능 같은 채널 상태는 고객별로 달라질 수 있으므로 접촉 이력과 같은 맥락에서 표시한다.

### 3.2 중앙 상단: 고객 전환 및 핵심 계좌 상태

고객 전환은 중앙 상단에서 수행한다.

표시 정보:

| 항목 | 내용 |
|---|---|
| 고객 인덱스 | `1 / 5` 형태, 좌우 화살표 포함 |
| 고객 선택 | 콤보박스, 예: `박재원(820312) / 1002-832` |
| 성별/연령 | 예: `44세(남)`, 성별 색상 구분 |
| 미납 금액 | 미납 잔액 |
| 현재 연체 | 현재 D+일수 |
| 최장 연체 | 최장 D+일수 |
| 누적 연체 | 총 연체일수 |
| 이율 | 연 이율 |
| 계좌 상태 | 정상/연체/상각 등 |

상단 정보는 한 줄 기준으로 정렬한다. 라벨이 두 줄로 깨지거나 카드 높이가 늘어나면 안 된다.

### 3.3 중앙 분석 라인

중앙 첫 번째 분석 라인은 3개 카드로 구성한다.

| 카드 | 표시명 | 역할 |
|---|---|---|
| 1 | 타임 라인 | 현재 회수 단계와 주요 일자 |
| 2 | 행동 분석 | 납부 패턴, 회수등급, 납부 일관성 |
| 3 | 감성 분석 | 감성 점수, 감성 구성 요소, 요약 |

3개 카드는 형식과 높이를 맞춘다. 각 카드에는 시각화 영역과 요약 영역을 균형 있게 배치한다.

### 3.4 타임 라인

타임 라인은 현재 계좌가 어떤 회수 절차 흐름 위에 있는지 보여준다.

표시 요소:

| 항목 | 내용 |
|---|---|
| 단계 도트 | 1~5 단계 진행 상태 |
| 현재 단계 | 예: `5단계`, `법적 단계` |
| 주요 일자 | 차기이수일, 최종상환일, 기한상실일 |
| 상세보기 | 단계별 실제 처리 이력 |

기한상실일은 실제 기한상실 대상인 고객에게만 빨간 강조를 적용한다. 기한상실 대상이 아니면 빨간 표시를 하지 않는다.

상세보기 모달은 고정 크기이며, 바깥 클릭으로 닫혀야 한다. 목록이 길 경우 전체 스크롤보다 펼쳐보기 중심으로 구성해 사용자 액션을 줄인다.

### 3.5 행동 분석

행동 분석은 “계좌 행동 분석”의 축약 화면이다.

표시 요소:

| 항목 | 내용 |
|---|---|
| 납부 패턴 | 예: 장기 미납 반복, 부분 연체 관찰, 정상 유지 |
| 회수등급 | A~E |
| 납부 일관성 | 최근 12개월 기준 정상 납부 비율 |
| 현재 판단 | 예: 법적조치 전환, 납부일 확약, 리마인드 유지 |

행동 분석은 액션 타임라인과 겹치면 안 된다. 타임라인이 “절차 상태”라면 행동 분석은 “납부 행동과 회수 가능성”을 보여준다.

### 3.6 감성 분석

감성 분석은 고객 반응과 민원 위험을 판단한다.

표시 요소:

| 항목 | 내용 |
|---|---|
| 감성 점수 | 0~100 |
| 원형 시각화 | 점수 구간에 따라 색상 변경 |
| 구성 지표 | 협조적 태도, 감정 안정도, 분할납부 수용 가능성 |
| 요약 | 고객 감성 상태를 1~2문장으로 요약 |

상세보기에는 점수 자체를 반복하기보다 “왜 이 점수가 나왔는지” 판단 근거를 보여준다.

### 3.7 상담 전략

상담 전략은 분석 결과의 결론 영역이다.

포함 내용:

| 항목 | 내용 |
|---|---|
| 고객 메모 | 상담원이 바로 참고할 개인 메모 |
| 행동 전략 | 회수/납부/절차 관점의 다음 행동 |
| 응대 전략 | 감성/민원/준법 관점의 말투 |
| 오늘 처리 | 상담원이 지금 해야 하는 핵심 액션 |
| 주의사항 | 확정 표현 금지, 압박 표현 금지, 개인정보 노출 금지 |

상담 전략에는 이미 상단/분석 카드에서 보이는 고객 특성을 반복 표시하지 않는다. 전략의 목적은 “그래서 지금 어떻게 응대할 것인가”를 보여주는 것이다.

### 3.8 우측 패널: AI스크립트

상단:

```text
AI스크립트
```

우측 라벨:

```text
행동전략 라벨 / 응대전략 라벨
```

구성:

| 블록 | 역할 |
|---|---|
| 예상 질문 | 고객별 preset 질문 4개 |
| 직접 입력 | 시연용 질문이 고객별로 자동 입력됨 |
| 호출 버튼 | 실제 LLM 호출 |
| 로딩 카드 | AI 호출 중, 경과 시간, 전략 반영/준법 점검/초안 정리 |
| 결과 카드 | 의도, 발송 가능 여부, 편집 가능한 스크립트, 다음 액션 |

예상 질문은 즉시 보여주는 시연 안정용이다. 하단 직접 입력은 실제 LLM 호출을 검증하는 용도다.

## 4. 고객 데이터 모델

현재 고객 객체는 아래 정보를 기준으로 구성한다.

| 필드 | 의미 |
|---|---|
| `id` | 고객 식별자 |
| `name` | 고객명 |
| `gender` | 남/여 |
| `birthDate` | 생년월일 |
| `product` | 햇살론/차량담보대출/신용대출/사잇돌 |
| `recoveryGrade` | 회수등급 A~E |
| `soundnessClass` | 정상/요주의/고정/회수의문/추정손실 |
| `traits` | 개인회생, 신용회복, 새출발기금, 경매, 채권보전, 민원 위험 등 |
| `blockedChannels` | 접촉 금지 채널 |
| `preferredChannels` | 우선 접촉 채널 |
| `interestRate` | 이율 |
| `loanAmount` | 대출 원금 |
| `paidAmount` | 납부 누계 |
| `overdueDays` | 현재 연체일수 |
| `maxOverdueDays` | 최장 연체일수 |
| `totalOverdueDays` | 누적 연체일수 |
| `accelerationDate` | 기한상실일 |
| `nextActionDate` | 차기이수일 |
| `finalPaymentDate` | 최종상환일 |
| `riskLevel` | none/observe/caution/warning/severe/danger |
| `step` | 회수 단계 1~5 |

## 5. 고객 분류 체계

### 5.1 회수 단계

| 단계 | 조건 | 화면명 | 기본 액션 |
|---|---|---|---|
| Step 1 | 정상 또는 D+0 인접 | 안내 단계 | 감사 안내, 다음 납부일 리마인드 |
| Step 2 | 단기 연체 | 독촉 단계 | 납부 가능일 확인 |
| Step 3 | D+30 이상 또는 반복 지연 | 경고 단계 | 분할납부 협의, 내용증명 전 안내 |
| Step 4 | D+60 이상 또는 조정/민원 위험 | 예고 단계 | 기한상실 예고, 채무조정 접수 확인 |
| Step 5 | D+90 이상 또는 법적 단계 | 법적 단계 | 기한상실 고지 이력 확인, 법적 절차 담당자 승인 |

### 5.2 회수등급

회수등급은 회수 가능성과 관리 강도를 나타내는 내부 분류다.

| 등급 | 의미 | 상담 방향 |
|---|---|---|
| A | 정상/우량 | 관계 유지, 리마인드 |
| B | 경미한 관찰 | 납부일 확인, 지연 방지 |
| C | 중위험 | 분할납부 협의, 납부 계획 확보 |
| D | 고위험 | 고지 이력 확인, 승인선 확인 |
| E | 회수 곤란 | 채무조정/송무/상각 가능성 검토 |

### 5.3 건전성 분류

| 분류 | 의미 | 화면 활용 |
|---|---|---|
| 정상 | 정상 관리 계좌 | 관계 유지 |
| 요주의 | 연체 또는 관찰 필요 | 행동 분석에서 주의 표시 |
| 고정 | 회수 위험 확정 구간 | 법적/담보/송무 확인 |
| 회수의문 | 회수 가능성 낮음 | 조정 가능성 및 상각 검토 |
| 추정손실 | 손실 가능성 높음 | 회수보다 사후 관리 중심 |

### 5.4 사고/송무 특성

사고 특성:

- 개인회생
- 신용회복
- 파산
- 새출발기금

송무 특성:

- 경매
- 민사집행
- 채권보전
- 집행권원
- 형사고소

이 특성은 상담 전략에서 반복 표시하지 않고, 계좌 특성 및 행동 분석의 판단 근거로 사용한다.

## 6. 점수화 기준

현재 점수화는 데모용 휴리스틱이다. 실제 운영에서는 과거 상담 결과, 회수율, 민원 발생률, 납부 약속 이행률로 보정해야 한다.

### 6.1 감성 점수

감성 점수는 0~100점이다.

| 구간 | 상태 | 응대 전략 |
|---|---|---|
| 0~39 | 위험 | 공감 우선, 압박 금지, 민원 예방 |
| 40~60 | 관찰 | 중립적 안내, 짧은 문장 |
| 61~74 | 협조 가능 | 구체적 선택지 제시 |
| 75~100 | 우호 | 감사/관계 유지 |

판단 입력:

- 최근 접촉 이력의 `sentimentScore`
- 상담 거부 표현
- 민원/항의/불만 키워드
- 법적 절차 언급에 대한 반응
- 채무조정/분할납부 수용 가능성

현재 코드 기준:

- 박재원: 18점 고정, 민원/거부 위험 고객
- 최미래: 92점 고정, 정상/우호 고객
- 그 외 고객: 최근 접촉 이력 평균

향후 개선:

```text
감성 점수 =
  접촉 이력 평균 40%
+ 부정 키워드 25%
+ 거부/민원 플래그 20%
+ 협조/상환의지 플래그 15%
```

### 6.2 행동 위험도

행동 위험도는 납부 행동과 연체 상태를 기준으로 `low / medium / high`로 분류한다.

현재 코드 기준:

```text
high:
  overdueDays >= 90
  또는 riskLevel == danger
  또는 미납 건수 >= 2

medium:
  overdueDays >= 30
  또는 riskLevel == warning
  또는 미납 건수 == 1

low:
  위 조건에 해당하지 않음
```

### 6.3 납부 일관성

납부 일관성은 최근 12개월 납부 이력 중 정상 납부 비율이다.

```text
납부 일관성 = 정상 납부 월수 / 전체 관찰 월수 * 100
```

표시 예:

```text
83% · 최근 12개월
정상 10 · 연체 1 · 미납 1
```

중복 방지 원칙:

- 상단: 현재/최장/누적 연체일수
- 행동 분석: 납부 패턴과 일관성
- 타임라인: 절차상 주요 일자

### 6.4 우선순위 분류

고객 처리 우선순위는 `critical / attention / routine`으로 분류한다.

```text
critical:
  D+90 이상
  또는 riskLevel == danger
  또는 감성 점수 < 25

attention:
  D+30 이상
  또는 감성 점수 < 61
  또는 미납 건수 > 0

routine:
  위 조건에 해당하지 않음
```

화면 표시:

| 우선순위 | 표시 의미 | 상담 방향 |
|---|---|---|
| critical | 오늘 최우선 | 절차 확인, 담당자 승인, 반복 독촉 지양 |
| attention | 상담 필요 | 납부 가능일/금액 확인 |
| routine | 관계 유지 | 감사 안내, 다음 약정일 리마인드 |

## 7. 분석 결과와 전략 매핑

### 7.1 1:1 매핑 원칙

화면은 아래 흐름으로 판단해야 한다.

```text
행동이 이렇다 → 상담 처리 방향은 이렇다
감성이 이렇다 → 상담 톤은 이렇다
계좌 특성이 이렇다 → 확인해야 할 근거/절차는 이렇다
```

### 7.2 행동 분석 → 행동 전략

| 행동 상태 | 전략 |
|---|---|
| 장기 미납 반복 | 고지 이력 확인, 법적 절차 승인선 확인 |
| 부분 연체 관찰 | 납부 가능일/가능 금액 확인 |
| 납부 이탈 심화 | 분할납부 또는 조정 가능성 검토 |
| 정상 유지 | 감사 안내, 다음 납부일 리마인드 |

### 7.3 감성 분석 → 응대 전략

| 감성 상태 | 톤 |
|---|---|
| 민원 위험형 | 절차 중심, 압박 금지 |
| 불안형 | 공감 우선, 확인 절차 분리 |
| 협조형 | 구체적 선택지 제시 |
| 우호형 | 감사/관계 유지 |

### 7.4 계좌 특성 → 확인 절차

| 특성 | 확인 항목 |
|---|---|
| 개인회생/신용회복/파산/새출발기금 | 접수 상태, 대상 채권 여부, 추심 진행 가능 여부 |
| 경매/민사집행/채권보전/집행권원 | 송무 담당자, 진행 단계, 고지 이력 |
| 차량담보대출 | 담보 상태, 회수 가능성, 송무 병행 여부 |
| 햇살론/사잇돌 | 보증기관 기준, 분할 협의 가능성 |

## 8. LLM 호출 전략

### 8.1 provider 구조

현재 서비스는 provider 순서를 env로 제어한다.

```env
LLM_PROVIDERS=gemini,groq,ollama
LLM_FALLBACK_ENABLED=true
```

로컬 테스트 모드:

```env
LLM_PROVIDERS=ollama
LLM_FALLBACK_ENABLED=false
LLM_OLLAMA_MODEL=gemma4:latest
```

### 8.2 provider 용도

| provider | 용도 | 특징 |
|---|---|---|
| Gemini 2.5 Flash | 시연/품질 우선 | 약 2초대, 전략 반영 안정적 |
| Groq | 빠른 fallback | 약 0.5초대, 문구는 짧고 단순 |
| Ollama gemma4 | 로컬 테스트 | 외부 API 없이 테스트 가능, 상대적으로 느림 |

Ollama `gemma4`는 `/api/chat`과 `think:false`를 사용해야 본문이 빈 값으로 떨어지는 문제를 줄일 수 있다.

### 8.3 호출 중 UI

실제 호출 대기 중에는 우측 AI스크립트 영역에 로딩 카드가 표시된다.

표시 내용:

- `AI 호출 중`
- 경과 시간
- 진행 바
- `전략 반영 / 준법 점검 / 초안 정리`

이 로딩은 가짜 응답을 보여주기 위한 것이 아니라 실제 provider 호출이 진행 중임을 나타내기 위한 상태다.

### 8.4 시연용 입력

고객마다 직접 입력창에 시연용 질문을 미리 넣는다.

| 고객 | 시연 질문 방향 |
|---|---|
| 박재원 | 법적조치 시점, 전화 금지, 문자 안내 |
| 오민석 | 새출발기금 접수, 추심 중단 여부, 이메일 안내 |
| 김철수 | 일부 납부, 분할납부 가능성 |
| 정서윤 | 급여일 후 납부 약속 |
| 최미래 | 자동이체 유지, 다음 납부일 확인 |
